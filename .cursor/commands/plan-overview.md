# plan-overview

Go over a plan created with another chat. 
Check for clarity, correctness and truthfulness.
Suggest any improvments that can make the plan and resulting code clearer, simpler, more maintainable and more robust. 
Give your suggestions in order of what you think is most important. 
Example of important things:
    - The plan will result in a solution to the discussed problem or refactoring
    - The plan solves the problem in the a maintainable way
    - The plan is easy to follow
    - The plan is comprehensive enough
    - etc.
